# Content

## 内容定位

博客以技术沉淀为主，不写泛泛日记。

主要记录：

- AI / Agent 使用经验
- Codex / Claude Code / OpenClaw 实践
- RPA / 影刀自动化
- Playwright / Web 自动化
- 项目方案与复盘
- 工具教程和排错记录

## 分类

| 分类 | 内容 |
|---|---|
| AI / Agent | AI 工具、Agent、模型、提示词 |
| RPA / 自动化 | 影刀、Playwright、自动化流程 |
| Projects | 项目展示、项目复盘 |
| Guides | 教程、工具使用、配置经验 |
| Diagrams | 流程图、架构图、图表示例 |

## 文章结构

普通文章建议：

```text
背景
→ 问题
→ 解决方案
→ 关键步骤 / 代码 / 流程图
→ 踩坑
→ 结论
```

项目文章建议：

```text
项目背景
→ 目标
→ 方案
→ 实现要点
→ 效果
→ 复盘
```

## 写作原则

- 标题直白。
- 一篇文章解决一个主题。
- 重要结论靠前。
- 代码和命令尽量可复制。
- 图表用于降低理解成本，不为了装饰硬加。
- 长文优先保证阅读节奏。

## Markdown / MDX

- 普通文章用 `.md`。
- 需要组件、交互或特殊展示时用 `.mdx`。
- Mermaid 图表直接写在文章中。
- 图片统一放到项目约定的静态资源目录后再引用。

## Mermaid 使用边界

适合：流程图、架构图、时序图、状态图、思维导图。

不适合：UI 原型图、复杂海报、大量文字说明图。

如果 Mermaid 表达成本太高，优先简化图，不急着引入新图表工具。
