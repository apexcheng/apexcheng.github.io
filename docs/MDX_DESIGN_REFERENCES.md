# MDX 文章视觉设计参考

这份文件不是组件 API 文档，而是给后续 Agent 的视觉判断参考。

本博客里的 MDX 可视化，优先服务个人笔记、实践复盘、经验文章和工具方案说明。目标不是把文章做成产品官网，而是让读者更快看见：结论是什么，问题在哪里，数据说明什么，下一步怎么判断。

## 总原则

1. 先判断文章要表达的结构，再选择视觉形式。
2. 视觉块要帮助读者扫描，不要替代正文思考。
3. 一个视觉块只表达一种关系：概览、指标、对比、路径、时间、案例或结论。
4. 表格只用于规则、字段、轻量参数对比；不要把长解释塞进表格。
5. 流程图只用于分支、依赖、回路或多角色关系；普通顺序不要默认画图。
6. 正式文章可以少用，但要用在关键处；视觉实验文章可以集中展示写法。

## Bento 宫格

Bento 宫格适合放文章开头的结构概览，或者把一个经验主题拆成几类重点。

适合：

- 影刀自动化方案的输入、动作、输出、风险。
- 电商经营看板的销售、退款、库存、推广。
- AI 工作流的触发、上下文、工具调用、人工确认。

不适合：

- 字段很多的清单。
- 每项都需要长篇解释的内容。
- 只有两个选项的简单对比。

常用写法：`VisualGrid` 加 `FeatureCard`，每张卡片一个角度。

## 指标卡片

指标卡片适合把数字、比例、状态、耗时、风险等级放出来，让读者先建立量感。

适合：

- 退款率、库存周转天数、滞销 SKU 数。
- 自动化节省时间、失败次数、人工确认节点。
- 文章方案的投入成本、可复用程度、维护风险。

不适合：

- 没有数字或状态的信息。
- 需要解释计算口径的复杂数据。
- 多行文本说明。

常用写法：`VisualGrid` 加 `MetricCard`，`value` 必须短，口径放在正文或 `detail`。

## 对比面板

对比面板适合两个或三个方案的判断，不只是字段对字段。

适合：

- 影刀普通版和企业版的边界。
- 手工日报和经营看板的差异。
- 表格记录、数据库、在线表格三种数据载体选择。

不适合：

- 只是参数清单。
- 每列都只有短词的对比。
- 多于三个方案且每项都很细。

常用写法：两列 `FeatureCard`。如果只是轻量参数，用 Markdown 表格更好。

## 时间线

时间线适合复盘过程，让读者看清事情是怎么演变的。

适合：

- 从手工统计到日报自动化，再到经营看板。
- 一次退款率异常的排查过程。
- 博客文章从素材、草稿、可视化到发布的过程。

不适合：

- 没有先后顺序的模块说明。
- 每一步都有复杂分支的流程。
- 纯任务清单。

常用写法：`DecisionFlow`。如果是日记式复盘，正文小标题也可以。

## 问题链路

问题链路适合展示一个业务问题如何一路传导到结果。

适合：

- 退款率升高：链接、型号、客服备注、库存批次的排查链。
- 库存周转变慢：销量、备货、促销、滞销款的关系。
- AI 工作流失败：输入缺失、上下文错误、工具返回异常、人工兜底。

不适合：

- 单一步骤的操作教程。
- 没有因果关系的列表。
- 只想展示漂亮路径的装饰图。

常用写法：简单链路用 `DecisionFlow`，有分支和回流时用 `Mermaid`。

## 结论卡片

结论卡片适合放文章开头或段落转折处，直接告诉读者这一节的判断。

适合：

- 文章开头的一句话结论。
- 某个方案的推荐结果。
- 风险提醒或发布前检查。

不适合：

- 连续堆多个重点，导致每个都不重点。
- 放普通段落。
- 只做装饰标题。

常用写法：`HighlightBox`。轻量补充用 `Callout`。

## 案例拆解卡

案例拆解卡适合把一个真实或模拟案例拆成背景、动作、结果、复盘。

适合：

- 一个店铺退款率异常的处理。
- 一条影刀流程从失败到稳定的调整。
- 一次 AI 工作流把日报变成看板的实践。

不适合：

- 没有案例，只是概念解释。
- 案例细节很多，需要连续叙述。
- 多个案例横向对比。

常用写法：`FeatureCard` 组合 3 到 4 张卡，或者一张 `HighlightBox` 加列表。

## 决策树

决策树适合帮助读者根据条件选择路径。

适合：

- 要不要用表格、卡片还是流程图。
- 影刀流程要不要上企业版。
- 电商数据异常先查哪个维度。

不适合：

- 没有明确条件的问题。
- 只是线性步骤。
- 判断条件太多，读者看不完。

常用写法：简单判断用 `DecisionFlow`，真正有分支用 `Mermaid`。

## 文章总结横幅

总结横幅适合在文章末尾收束，让读者带走一个可执行判断。

适合：

- “下次遇到同类问题，先做哪三件事”。
- “这套方案适合什么，不适合什么”。
- “保留什么边界，避免什么误用”。

不适合：

- 重复全文所有内容。
- 放太长的操作步骤。
- 引入新观点。

常用写法：`HighlightBox`，内容控制在 1 到 2 段。

## 本博客常见主题的选择建议

| 主题 | 优先视觉形式 | 少用什么 |
| --- | --- | --- |
| 影刀 RPA 流程复盘 | 时间线、问题链路、案例拆解卡 | 大表格 |
| 电商经营看板 | 指标卡片、Bento 宫格、对比面板 | 复杂流程图 |
| 库存周转分析 | 指标卡片、问题链路、决策树 | 纯文字长列表 |
| 退款率排查 | 问题链路、案例拆解卡、指标卡片 | 只有表格 |
| AI 工作流 | 问题链路、决策树、时间线 | 无意义卡片堆叠 |
| 博客项目说明 | Bento 宫格、时间线、结论卡片 | 过大的架构图 |

## 最小选型顺序

```text
先问：这段内容是数字、对比、步骤、因果、案例，还是结论？
数字 -> MetricCard
同类概览 -> VisualGrid + FeatureCard
两个方案取舍 -> 对比面板
线性过程 -> DecisionFlow
因果或分支 -> Mermaid
文章级判断 -> HighlightBox
普通叙述 -> Markdown
```
