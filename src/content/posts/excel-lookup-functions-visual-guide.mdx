`VLOOKUP`、`XLOOKUP`、`INDEX`、`MATCH`、`XMATCH` 经常被放在一起学习，但它们并不是五种完全不同的做法。

先记住最重要的关系：**VLOOKUP 和 XLOOKUP 是一步返回结果；MATCH 和 XMATCH 负责找到位置；INDEX 再根据位置取回内容。**

      <p>适合日常按 SKU、订单号或员工编号返回名称、价格、平台等数据。</p>

## 统一案例：根据 SKU 查商品信息

整篇文章使用同一张商品资料表。现在要查找 SKU `K161`，并返回商品名称或价格。

        <tr><td>3</td><td>M30</td><td>无线鼠标</td><td>京东</td><td>99</td></tr>
        <tr><td>4</td><td>P12</td><td>平板保护套</td><td>拼多多</td><td>139</td></tr>

这里的“位置 2”是 `K161` 在查找区域 `B2:B5` 中排第 2 个，不是 Excel 工作表的真实行号。这个区别对理解 `MATCH`、`XMATCH` 和 `INDEX` 很重要。

## 五个函数分别负责什么

<div class="lookup-role-map" role="group" aria-label="VLOOKUP XLOOKUP INDEX MATCH XMATCH 的功能分工">
  <article>
    <span>直接返回</span><strong>VLOOKUP</strong><p>在区域最左列查找，再按列序号向右返回。</p><small>老牌 · 兼容广</small>
  </article>
  <article>
    <span>直接返回</span><strong>XLOOKUP</strong><p>查找区域和返回区域分开写，方向更灵活。</p><small>直观 · 推荐优先</small>
  </article>
  <article>
    <span>返回位置</span><strong>MATCH</strong><p>告诉你目标在区域里排第几个，精确匹配通常写 0。</p><small>位置函数</small>
  </article>
  <article>
    <span>返回位置</span><strong>XMATCH</strong><p>MATCH 的现代写法，默认就是精确匹配。</p><small>位置函数</small>
  </article>
  <article>
    <span>按位置取值</span><strong>INDEX</strong><p>你给它一个位置，它从指定区域中返回对应内容。</p><small>常与 MATCH 组合</small>
  </article>
</div>

## VLOOKUP：在最左列查找，再向右返回

要根据 G2 中的 `K161` 返回价格，可以写：

```text
=VLOOKUP(G2,B2:E5,4,FALSE)
```

<div class="lookup-formula-board" role="group" aria-label="VLOOKUP 公式参数与查找过程">
  <div class="lookup-formula-line">
    <span>=VLOOKUP(</span><b class="tone-orange">G2</b><span>,</span><b class="tone-blue">B2:E5</b><span>,</span><b class="tone-violet">4</b><span>,</span><b class="tone-green">FALSE</b><span>)</span>
  </div>
  <div class="lookup-formula-parts">
    <div class="tone-orange"><small>查找谁</small><strong>G2：K161</strong></div>
    <div class="tone-blue"><small>在哪查</small><strong>B2:E5</strong></div>
    <div class="tone-violet"><small>返回哪列</small><strong>区域第 4 列</strong></div>
    <div class="tone-green"><small>怎么匹配</small><strong>FALSE：精确</strong></div>
  </div>
  <div class="lookup-formula-path">
    <span>在 B 列找到 K161</span><i>→</i><span>命中区域第 2 行</span><i>→</i><strong>从第 4 列返回 199</strong>
  </div>
</div>

VLOOKUP 的关键限制是：**查找列必须位于所选区域的最左侧，返回值只能来自右边。** 此外，返回列依赖数字 `4`，表格列较多时容易数错。

## XLOOKUP：查找列和返回列直接对应

相同需求使用 XLOOKUP：

```text
=XLOOKUP(G2,B2:B5,E2:E5,"未找到")
```

<div class="lookup-formula-board is-xlookup" role="group" aria-label="XLOOKUP 公式参数与查找过程">
  <div class="lookup-formula-line">
    <span>=XLOOKUP(</span><b class="tone-orange">G2</b><span>,</span><b class="tone-blue">B2:B5</b><span>,</span><b class="tone-violet">E2:E5</b><span>,</span><b class="tone-green">{'"未找到"'}</b><span>)</span>
  </div>
  <div class="lookup-formula-parts">
    <div class="tone-orange"><small>查找谁</small><strong>G2：K161</strong></div>
    <div class="tone-blue"><small>在哪查</small><strong>SKU 列</strong></div>
    <div class="tone-violet"><small>从哪返回</small><strong>价格列</strong></div>
    <div class="tone-green"><small>找不到时</small><strong>显示“未找到”</strong></div>
  </div>
  <div class="lookup-formula-path">
    <span>SKU 列找到 K161</span><i>→</i><span>保持同一行</span><i>→</i><strong>从价格列返回 199</strong>
  </div>
</div>

XLOOKUP 不需要数列号。查找区域和返回区域分开后，公式读起来就是：**在 SKU 列找 G2，再从价格列返回结果。**

它也可以向左返回。例如根据商品名称返回 SKU：

```text
=XLOOKUP("蓝牙键盘",C2:C5,B2:B5,"未找到")
```

<div class="lookup-compare-board" role="group" aria-label="VLOOKUP 和 XLOOKUP 的主要区别">
  <section>
    <span>VLOOKUP</span>
    <strong>用一个大区域和列序号</strong>
    <div><b>查找列</b><i>→</i><b>第 2 / 3 / 4 列</b></div>
    <ul>
      <li>只能从区域最左列开始查找。</li>
      <li>返回列通过数字指定。</li>
      <li>适合旧表格和兼容场景。</li>
    </ul>
  </section>
  <section class="is-recommended">
    <span>XLOOKUP</span>
    <strong>查找区域和返回区域分开</strong>
    <div><b>SKU 列</b><i>↔</i><b>任意返回列</b></div>
    <ul>
      <li>可以向左或向右返回。</li>
      <li>不用计算返回列是第几列。</li>
      <li>当前版本支持时通常更容易维护。</li>
    </ul>
  </section>
</div>

## MATCH 和 XMATCH：只返回“第几个”

MATCH 和 XMATCH 不会直接返回商品名称或价格，它们只负责定位。

```text
=MATCH(G2,B2:B5,0)
```

```text
=XMATCH(G2,B2:B5)
```

两条公式都会返回 `2`。

<div class="lookup-position-board" role="group" aria-label="MATCH 和 XMATCH 返回相对位置 2">
  <header><span>查找区域 B2:B5</span><strong>目标：K161</strong></header>
  <div class="lookup-position-ruler">
    <span><small>1</small><b>X51C</b></span>
    <span class="is-hit"><small>2</small><b>K161</b></span>
    <span><small>3</small><b>M30</b></span>
    <span><small>4</small><b>P12</b></span>
  </div>
  <div class="lookup-position-results">
    <article><code>MATCH(G2,B2:B5,0)</code><strong>2</strong><p>第三个参数写 0，表示精确匹配。</p></article>
    <article><code>XMATCH(G2,B2:B5)</code><strong>2</strong><p>默认使用精确匹配，写法更短。</p></article>
  </div>
</div>

这两个函数最容易被误解的地方是：返回的 `2` 只是目标在当前查找区域中的相对位置。

## INDEX：根据位置取回内容

INDEX 的作用很直接：从一个区域中取第几个值。

```text
=INDEX(C2:C5,2)
```

在商品名称列 `C2:C5` 中，第 2 个值是 `蓝牙键盘`，因此公式返回蓝牙键盘。

真正实用的写法，是让 MATCH 或 XMATCH 动态提供这个位置。

<div class="lookup-composition-board" role="group" aria-label="INDEX 与 MATCH 或 XMATCH 组合查找">
  <section>
    <span>第一步 · 定位</span>
    <code>{'=MATCH(G2,B2:B5,0)'}</code>
    <strong>返回位置 2</strong>
  </section>
  <i>→</i>
  <section>
    <span>第二步 · 取值</span>
    <code>{'=INDEX(C2:C5,2)'}</code>
    <strong>返回蓝牙键盘</strong>
  </section>
  <i>→</i>
  <section class="is-complete">
    <span>组合公式</span>
    <code>{'=INDEX(C2:C5,MATCH(G2,B2:B5,0))'}</code>
    <strong>动态完成查找</strong>
  </section>
</div>

使用 XMATCH 时，只需要把中间的 MATCH 替换掉：

```text
=INDEX(C2:C5,XMATCH(G2,B2:B5))
```

这种组合的理解方式是：

```text
MATCH / XMATCH：K161 在第几个？
INDEX：那就取商品名称列的第几个。
```

## MATCH 和 XMATCH 怎么选

<div class="lookup-compare-board is-position-compare" role="group" aria-label="MATCH 和 XMATCH 的选择对比">
  <section>
    <span>MATCH</span>
    <strong>经典位置函数</strong>
    <ul>
      <li>精确匹配通常明确写第三个参数 `0`。</li>
      <li>旧文件和现有公式中很常见。</li>
      <li>适合需要兼容旧写法的场景。</li>
    </ul>
  </section>
  <section class="is-recommended">
    <span>XMATCH</span>
    <strong>更直观的位置函数</strong>
    <ul>
      <li>默认就是精确匹配。</li>
      <li>和 XLOOKUP 的参数思路更接近。</li>
      <li>当前版本支持时，写新公式更简洁。</li>
    </ul>
  </section>
</div>

## 最后到底怎么选

<div class="lookup-choice-board" role="group" aria-label="五个查找匹配函数的选择建议">
  <article><span>日常直接返回结果</span><strong>XLOOKUP</strong><p>版本支持时优先考虑，公式最容易读懂。</p></article>
  <article><span>旧文件或兼容优先</span><strong>VLOOKUP</strong><p>适合查找列在左、返回列在右的普通表格。</p></article>
  <article><span>只需要目标位置</span><strong>XMATCH / MATCH</strong><p>返回第几个，不直接返回名称或金额。</p></article>
  <article><span>定位和取值分开</span><strong>INDEX + XMATCH</strong><p>逻辑清楚，也可以使用 INDEX + MATCH。</p></article>
</div>

## 五个容易出错的地方

<div class="lookup-warning-stack" role="group" aria-label="查找匹配函数常见错误">
  <article><span>01</span><div><strong>VLOOKUP 忘记写 FALSE</strong><p>做普通精确查找时，最后一个参数应明确写 `FALSE`。</p></div><b>匹配方式</b></article>
  <article><span>02</span><div><strong>把 VLOOKUP 的列序号当成工作表列号</strong><p>数字 4 表示所选区域中的第 4 列，不是 Excel 的 D 列。</p></div><b>区域相对位置</b></article>
  <article><span>03</span><div><strong>把 MATCH 返回值当成真实行号</strong><p>返回 2 表示在查找区域中排第 2 个，不代表工作表第 2 行。</p></div><b>相对位置</b></article>
  <article><span>04</span><div><strong>INDEX 区域和查找区域没有对齐</strong><p>两段区域的顺序和长度应保持对应，否则位置 2 可能取到另一条数据。</p></div><b>区域对应</b></article>
  <article><span>05</span><div><strong>忽略重复查找值</strong><p>存在多个相同 SKU 时，普通正向查找通常先返回第一条匹配记录。</p></div><b>数据唯一性</b></article>
</div>

<HighlightBox eyebrow="记忆方法" title="不要背五套逻辑，只记两条路线">
  <p><strong>一步返回：</strong>VLOOKUP、XLOOKUP。</p>
  <p><strong>先定位再取值：</strong>MATCH / XMATCH 找位置，INDEX 按位置返回内容。</p>
</HighlightBox>
