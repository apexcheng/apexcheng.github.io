很多人在使用 AI 助手时都会遇到一个问题：

> 昨天已经告诉它的事情，今天为什么又忘了？

原因很简单：大部分 AI 对话只有当前窗口的短期记忆。

当任务变成长期项目，例如维护博客、开发 Agent、管理自动化流程时，仅靠聊天上下文已经不够了。

所以我尝试给 OpenClaw 增加一个长期记忆系统，让它能够：

- 记住长期项目背景；
- 保存过去的重要决策；
- 在需要的时候自动回忆相关信息；
- 随着使用时间增加不断积累经验。

这套方案的核心就是：**OpenClaw 负责思考和执行，GBrain 负责长期记忆。**

<HighlightBox eyebrow="一句话理解" title="让 AI 从临时助手变成长期合作伙伴">
  普通 AI 像一个能力很强但刚入职的员工，每次聊天都需要重新介绍背景。接入长期记忆后，它更像一个持续合作的同事。
</HighlightBox>

## OpenClaw 和 GBrain 分别负责什么？

不要把 GBrain 理解成另一个聊天机器人。

它更像是 Agent 的知识库和长期记忆层。

<Mermaid
  title="OpenClaw + GBrain 的关系"
  chart={`flowchart TD
    U[用户] --> O[OpenClaw Agent]

    G --> E[经验总结]

/>

简单理解：

| 部分 | 作用 |
| --- | --- |
| OpenClaw | AI 助手的大脑，负责推理、调用工具、执行任务 |
| GBrain | 长期记忆，保存项目、经验和历史信息 |
| Memory Core | OpenClaw 自己的短期状态 |

## 为什么 MCP 不等于长期记忆？

刚开始接入时，我也认为：

> 安装 GBrain MCP，让 Agent 可以调用工具，不就完成了吗？

实际并不是。

MCP 解决的是：

> Agent 可以主动查询和写入记忆。

但真正的长期记忆还需要两个能力：

### 1. 自动想起来

当用户提到某个项目时，系统应该主动找到相关历史信息，而不是等待 Agent 手动搜索。

这部分由 Context Engine 负责。

### 2. 自动记下来

聊天过程中，并不是所有内容都值得保存。

例如：

```text
今天帮我改一下标题
```

没有必要长期保存。

但：

```text
以后博客文章默认使用视觉晶蓝主题
```

这种长期规则就应该记录。

这部分由 Signal Detector 判断。

## 最终架构

我的最终方案不是替换 OpenClaw 原有记忆，而是分层：

<Mermaid
  title="短期记忆和长期记忆分工"
  chart={`flowchart LR
    U[用户消息] --> O[OpenClaw]
    O --> M[短期记忆 Memory Core]
    O --> C[GBrain Context Engine]
    C --> R[自动召回历史知识]
    U --> S[Signal Detector]
    S --> J{是否值得保存}
    J -->|是| G[GBrain]
    J -->|否| X[忽略]`}
/>

这样：

- 当前聊天状态继续交给 OpenClaw；
- 长期知识交给 GBrain；
- 重要经验自动沉淀。

## 实际使用场景

### 场景一：维护博客

以前：

```text
每次开始写文章：
告诉 AI 我的博客风格、目录结构、视觉要求……
```

现在：

```text
AI 已经知道：
- 博客定位
- 写作习惯
- 文章视觉规范
- 过去做过的项目
```

沟通成本明显降低。

### 场景二：长期开发项目

大型项目最怕的不是代码，而是历史决策丢失。

例如：

```text
为什么不用方案 A？
之前遇到什么问题？
这个设计是谁决定的？
```

长期记忆可以保存这些上下文。

## 我的配置过程

最终配置主要分几个步骤：

### 1. 使用独立知识仓库

我没有把长期知识直接放进 OpenClaw 工作目录，而是单独维护：

```text
OpenClaw workspace
    ↓
Agent 配置、Skill、运行环境

GBrain knowledge
    ↓
长期知识、项目经验、历史记录
```

这样两个系统职责更清晰。

### 2. 接入 Context Engine

只安装 MCP 后，GBrain 只是一个工具。

接入 Context Engine 后，它才真正参与每轮对话上下文生成。

### 3. 增加 Signal Detector

通过消息触发机制，在后台判断哪些内容值得保存。

主 Agent 继续正常回复，不需要等待记忆写入完成。

## 遇到的几个坑

### MCP 可以用，但 Agent 不会自动回忆

原因：

MCP 只是工具，不负责自动注入上下文。

解决：

启用 Context Engine。

---

### Skill 写了触发规则，但没有自动执行

原因：

Skill 文件只是说明能力，不等于系统事件。

解决：

使用 OpenClaw 的事件机制触发 Signal Detector。

---

### 长期记忆不是越多越好

如果什么都保存，最后得到的是垃圾堆。

真正有价值的是：

- 稳定偏好；
- 长期项目背景；
- 重要决策；
- 可复用经验。

## 最后的体验

接入 GBrain 后，最大的变化不是 AI 变得“更聪明”。

而是减少了重复沟通。

以前像：

> 一个能力很强但每次见面都需要重新介绍自己的助手。

现在更像：

> 一个长期合作，逐渐了解你的 AI 同事。

这也是我认为 Agent 下一阶段很重要的方向：

**不仅要会执行任务，还要能够长期积累经验。**
