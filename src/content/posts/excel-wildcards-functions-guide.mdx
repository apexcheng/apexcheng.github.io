在 Excel 里，通配符就是用一个符号代替不确定的文字。比如商品编码可能是 `K161-黑色`、`K161-白色`、`K161-蓝色`，只要共同点是以 `K161` 开头，就不必先用 `LEFT` 截取，可以直接用 `"K161*"` 匹配。

但并不是所有函数都能直接识别通配符。即使支持，不同函数的开启方式也不一样：条件统计函数直接写，`XLOOKUP` 和 `XMATCH` 要把匹配模式设为 `2`，`VLOOKUP`、`HLOOKUP` 和 `MATCH` 则要使用精确匹配参数。

## Excel 只有三种通配符

严格来说，真正负责模糊匹配的是 `*` 和 `?`；`~` 是转义符，用于告诉 Excel：后面的星号或问号是普通字符，不要把它当作通配符。

    </ul>
  </FeatureCard>
  <FeatureCard title="~" meta="取消通配" description="查找真正的星号或问号时，写在符号前面。">
    <ul>

    </ul>

| 写法 | 匹配规则 | 可以匹配 | 不会匹配 |
| --- | --- | --- | --- |
| `K161*` | 以 K161 开头 | `K161`、`K161-黑色` | `黑色-K161` |
| `*K161` | 以 K161 结尾 | `黑色-K161` | `K161-黑色` |
| `*K161*` | 任意位置包含 K161 | `黑色-K161-套装` | `K162-黑色` |
| `K161?` | K161 后正好一个字符 | `K161A`、`K161黑` | `K161`、`K161黑色` |
| `K?61` | K 和 61 中间正好一个字符 | `K161`、`KA61` | `K1261` |
| `~*` | 查找真正的星号 | `赠品*` 中的 `*` | 普通文字 |

### 支持方式速查表

| 函数 | 通配符写在哪里 | 必须注意的参数 | 常见用途 |
| --- | --- | --- | --- |
| `COUNTIF`、`COUNTIFS` | 条件参数 | 无 | 统计某类编码、标题或状态数量 |
| `SUMIF`、`SUMIFS` | 条件参数 | 无 | 汇总某类商品的销量或金额 |
| `AVERAGEIF`、`AVERAGEIFS` | 条件参数 | 无 | 计算某类商品的平均值 |
| `MAXIFS`、`MINIFS` | 条件参数 | 无 | 找出某类数据的最大值或最小值 |
| `VLOOKUP`、`HLOOKUP` | 查找值 | 最后一个参数写 `FALSE` | 返回第一条符合模糊条件的数据 |
| `MATCH` | 查找值 | 第三个参数写 `0` | 返回第一条匹配记录的位置 |
| `XLOOKUP` | 查找值 | 第五个参数 `match_mode` 写 `2` | 返回第一条符合通配符的数据 |
| `XMATCH` | 查找值 | 第三个参数 `match_mode` 写 `2` | 返回第一条符合通配符的位置 |
| `SEARCH` | `find_text` 参数 | 无 | 在单元格文字内部查找模式 |

## 三类函数的写法不一样

### 1. 条件统计函数：直接写通配符

这类函数最简单，只需要把通配符写进条件中。

```text
=COUNTIF(C8:C17,"K161*")
```

统计 C8:C17 中所有以 `K161` 开头的单元格数量。

```text
=SUMIF(C8:C17,"K161*",D8:D17)
```

找到 C 列中以 `K161` 开头的记录，再汇总对应 D 列。

```text
=AVERAGEIF(C8:C17,"K161*",D8:D17)
```

找到 C 列中以 `K161` 开头的记录，再计算对应 D 列平均值。

条件来自单元格时，不要把单元格引用放进引号：

```text
=AVERAGEIF(C8:C17,F2&"*",D8:D17)
```

假设 F2 是 `K161`，`F2&"*"` 会拼成 `K161*`。

### 2. 查找函数：先开启正确的匹配模式

`XLOOKUP` 默认是精确匹配，但默认不会把 `*` 和 `?` 当成通配符。要使用通配符，必须把第五个参数设为 `2`：

```text
=XLOOKUP("K161*",C8:C17,D8:D17,"未找到",2)
```

它会返回第一条以 `K161` 开头的记录所对应的 D 列结果。

`XMATCH` 同样需要把第三个参数设为 `2`：

```text
=XMATCH("K161*",C8:C17,2)
```

传统查找函数使用精确匹配参数时，也能识别通配符：

```text
=VLOOKUP("K161*",A2:D100,4,FALSE)
```

```text
=MATCH("K161*",C8:C17,0)
```

<HighlightBox eyebrow="关键区别" title="精确匹配，不等于不能使用通配符">
  对 VLOOKUP、HLOOKUP 和 MATCH 来说，FALSE 或 0 的意思是不要做近似区间查找。查找值中如果包含 * 或 ?，Excel 仍然会按通配符规则匹配。XLOOKUP 和 XMATCH 的设计不同，需要专门选择 match_mode = 2。
</HighlightBox>

### 3. SEARCH：在文本内部查找模式

`SEARCH` 返回的是匹配内容在文本中的起始位置。比如编码中间有一个字符不确定：

```text
=SEARCH("K?61",C8)
```

如果 C8 是 `K161-黑色`，结果为 `1`。如果找不到，会返回 `#VALUE!`。

实际判断时，通常会和 `ISNUMBER` 组合：

```text
=ISNUMBER(SEARCH("K?61",C8))
```

找到时返回 `TRUE`，找不到时返回 `FALSE`。

## 回到最开始的 AVERAGEIF 公式

原来的公式是：

```text
=AVERAGEIF(MAP(LEFT(C8:C17,4),LAMBDA(r,LEFT(r,4))),"K161",D8:D17)
```

它的真实目标只是：**找出 C 列中所有以 K161 开头的记录，再计算 D 列平均值。**

因此可以直接改成：

```text
=AVERAGEIF(C8:C17,"K161*",D8:D17)
```

新写法不需要 `LEFT`、`MAP` 和 `LAMBDA`，也不需要先生成中间数组。条件本身就是“以 K161 开头”，用原生通配符更贴近实际需求。

## 容易踩的五个坑

### 1. XLOOKUP 没写匹配模式 2

下面的公式不会启用通配符匹配：

```text
=XLOOKUP("K161*",C8:C17,D8:D17)
```

应该写成：

```text
=XLOOKUP("K161*",C8:C17,D8:D17,"未找到",2)
```

### 2. `*` 不等于“所有非空单元格”

`COUNTIF(A2:A100,"*")` 主要用于统计包含文本的单元格，不能简单理解为统计所有非空内容。需要统计数字、文字和日期等所有非空单元格时，优先使用：

```text
=COUNTA(A2:A100)
```

### 3. 通配符匹配通常不区分英文大小写

`K161*` 和 `k161*` 通常会匹配到相同内容。需要严格区分英文大小写时，不能只依赖通配符条件。

### 4. FIND 和 FILTER 不会直接解释通配符

`FIND` 不是通配符搜索函数；`FILTER` 的筛选条件也必须是一组 `TRUE/FALSE`。需要按包含关系筛选时，可以组合 `SEARCH`：

```text
=FILTER(D8:D17,ISNUMBER(SEARCH("K161",C8:C17)))
```

### 5. 前后空格仍然会影响匹配位置

`K161*` 要求内容从 K161 开始。如果单元格实际是 ` K161-黑色`，前面多了空格，就不会被视为以 K161 开头。数据来源不干净时，先检查是否需要 `TRIM` 或清洗源数据。
