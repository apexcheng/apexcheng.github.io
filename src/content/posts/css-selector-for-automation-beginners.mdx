刚开始学习网页自动化时，最先遇到的问题通常不是“怎么点击”，而是：**怎么告诉程序，应该点击页面上的哪一个按钮？**

浏览器里可能同时有几十个按钮、输入框和表格项。自动化程序必须先准确找到目标元素，才能继续点击、输入或读取数据。这个过程就是**元素定位**。

对于刚入门的实习生，我更建议先学习 CSS：语法短、和网页结构接近、浏览器可以直接验证，大部分常规页面也已经够用。

## 先看懂网页元素有什么特征

一个登录输入框可能长这样：

```html
<input
  id="username"
  class="login-input"
  name="username"
  placeholder="请输入用户名"
  type="text"
/>
```

可以用来定位它的特征包括：

| 特征 | 值 | CSS 写法 |
| --- | --- | --- |
| 标签 | `input` | `input` |
| id | `username` | `#username` |
| class | `login-input` | `.login-input` |
| name | `username` | `input[name="username"]` |
| placeholder | `请输入用户名` | `input[placeholder="请输入用户名"]` |
| type | `text` | `input[type="text"]` |

CSS 定位做的事情，就是从这些特征中选择一个或多个，组合成能找到目标元素的表达式。

### 1. 通过 id 定位

```html
<button id="submit-order">提交订单</button>
```

```css
#submit-order
```

在 Playwright 中可以直接点击：

```python
page.locator("#submit-order").click()
```

`id` 理论上应该在页面中唯一，但要留意动态生成的值：

```html
<button id="button_183746291">提交</button>
```

如果刷新页面后数字会变化，这个 id 看起来很具体，实际上并不稳定。

### 2. 通过 class 定位

```html
<button class="button primary large">提交</button>
```

可以写成：

```css
.primary
.button.primary
```

这里最容易混淆的是空格：

```css
.button.primary  /* 同一个元素同时拥有两个 class */
.button .primary /* 在 .button 里面寻找 .primary 子元素 */
```

### 3. 通过属性定位

属性定位在实际自动化里非常常用：

```css
input[name="username"]
input[placeholder="请输入订单号"]
button[data-action="submit"]
button[aria-label="关闭"]
```

类似 `name`、`data-testid`、`data-action`、`aria-label` 的属性通常比颜色、位置和随机 class 更适合长期维护，因为它们更接近元素的功能。

### 4. 组合多个条件

一个条件不能唯一定位时，再组合：

```css
input.form-input[name="keyword"]
```

它表示：

- 标签是 `input`；
- class 包含 `form-input`；
- name 等于 `keyword`。

条件不是越多越好。下面这种完整路径虽然具体，但页面只要调整一层结构就可能失效：

```css
body .page .content .form div input.form-input[name="keyword"]
```

真正的目标是：**用尽量少、但足够稳定的条件，唯一找到目标元素。**

### 5. 先找区域，再找子元素

页面上可能同时有多个“确认”按钮。这时不要只写：

```css
.confirm
```

而是先限定弹窗范围：

```css
.dialog .confirm
```

空格表示在 `.dialog` 内部寻找 `.confirm`。自动化中经常会先限定弹窗、表格行、商品卡片或某个表单，再找里面的按钮。

### 6. 按位置定位

```css
tbody tr:nth-child(2) td:nth-child(3)
```

`nth-child(2)` 表示父元素中的第二个子元素。它可以用于临时测试，但不应该成为首选，因为页面新增一行数据后，原来的第二行就可能变成第三行。

自动化最危险的情况不是“找不到”，而是**找到了错误的元素，并继续执行**。

写 CSS 时不要一上来复制浏览器生成的超长路径。先按下面四步判断，定位通常会更短，也更容易维护。

## 怎么在浏览器里查看和验证？

### 第一步：打开开发者工具

在 Chrome 或 Edge 中按 `F12`，或者在目标元素上点击右键，选择“检查”。

### 第二步：观察元素特征

重点看这些内容：

- 标签名；
- id；
- class；
- name；
- placeholder；
- data 属性；
- aria-label。

不要看到一大段 HTML 就慌。先问自己：有没有稳定 id？有没有能表达功能的属性？是否需要先限定一个区域？

### 第三步：在 Console 验证

```javascript
document.querySelectorAll('input[name="username"]')
```

重点看返回数量：

| 数量 | 说明 | 下一步 |
| --- | --- | --- |
| 0 个 | 没找到，或者元素还没有加载 | 检查语法、页面状态和 iframe |
| 1 个 | 通常是理想结果 | 再确认它是不是目标元素 |
| 多个 | 定位范围太大 | 增加区域或稳定条件 |

一个好的定位不能只看“现在能找到”，还要看“以后是否容易失效”。

## CSS 本身不能按文字定位，影刀里怎么做？

标准 CSS 不能根据元素内部显示的文字进行匹配。

例如页面上有一个按钮：

```html
<button>提交订单</button>
```

下面这种写法并不是有效的 CSS 文本定位：

```css
button[text="提交订单"]
```

也不要在影刀中写：

```css
button:contains("提交订单")
```

`:contains()` 不是浏览器支持的标准 CSS 选择器。影刀的 `find_by_css()` 应该按照普通 CSS 选择器使用。

### 方法一：使用 XPath 按文字定位

在影刀编码版中，需要按照文字查找元素时，可以直接使用 XPath。

精确匹配按钮文字：

```python
button = browser.find_by_xpath(
    '//button[normalize-space(.)="提交订单"]',
    timeout=10,
)

button.click()
```

`normalize-space(.)` 会清理文字前后的空格和换行。

例如下面的按钮：

```html
<button>
  提交订单
</button>
```

仍然可以被正常找到。

如果按钮文字还包含其他内容，可以使用模糊匹配：

```python
button = browser.find_by_xpath(
    '//button[contains(normalize-space(.), "提交订单")]',
    timeout=10,
)

button.click()
```

这会匹配：

```text
提交订单
立即提交订单
确认提交订单
```

因此，能够精确匹配时，优先使用精确匹配，避免找到错误按钮。

### 方法二：先用 CSS 缩小范围，再读取文字

有时页面结构适合使用 CSS，但目标元素没有稳定属性。

可以先通过 CSS 找到一组候选按钮：

```python
buttons = browser.find_all_by_css(
    ".order-dialog button",
    timeout=10,
)
```

再读取每个元素的文字：

```python
submit_button = None

for button in buttons:
    if button.get_text().strip() == "提交订单":
        submit_button = button
        break

if submit_button is None:
    raise RuntimeError('未找到文本为“提交订单”的按钮')

submit_button.click()
```

这种方式的思路是：

```text
CSS 缩小查找范围
        ↓
get_text() 读取元素文字
        ↓
Python 判断是不是目标元素
```

它确实能够按文字找到元素，但需要注意：

> 文字判断发生在 Python 代码中，并不是 CSS 选择器本身支持文字定位。

### 能用稳定属性时，仍然优先使用 CSS

如果按钮本身有稳定属性：

```html
<button data-action="submit-order">提交订单</button>
```

直接使用 CSS 会更简单：

```python
button = browser.find_by_css(
    'button[data-action="submit-order"]',
    timeout=10,
)

button.click()
```

在影刀中可以按照下面的顺序选择：

```text
有稳定属性
    ↓
优先使用 CSS

没有稳定属性，但文字明确
    ↓
使用 XPath 按文字定位

需要先限定弹窗、表格或卡片范围
    ↓
CSS 找候选元素，再通过 get_text() 筛选
```

学习 CSS 并不代表所有元素都必须使用 CSS。

真正的目标始终是：**选择最清楚、最稳定、最容易维护的定位方式。**
