## 代表题型

完整题目请下载附件。下面挑选四类题目，说明这套考核具体在检查什么。

### 类型转换不是字符串拼接

```python
price = "119.9"
quantity = "2"

# 计算订单总金额，并输出：总金额: 239.8
```

这道题很简单，但可以直接看出考核者是否知道外部输入经常是字符串，以及数值计算前需要先转换类型。

### 字典统计是否能独立写出来

```python
products = ["键盘", "鼠标", "键盘", "耳机", "鼠标", "键盘"]

# 期望输出：
# {'键盘': 3, '鼠标': 2, '耳机': 1}
```

这道题不限定必须使用哪种高级写法。重点是能否用循环、判断和字典完成一个完整统计过程。

### 文件题要和附件一起使用

```python
# 请读取 test.txt 文件内容并打印。
```

`test.txt` 已作为第二个附件提供。把它和 Python 文件放在同一目录，就可以检查相对路径、文件打开和内容读取这些基础操作。

### 最后三题不允许运行代码

```python
original = {
    "name": "张三",
    "skills": ["Python", "SQL"]
}

copy_data = original.copy()
copy_data["name"] = "李四"
copy_data["skills"].append("Linux")
```

这里不是考察是否会背 `copy()`，而是看考核者能否区分：外层字典已经复制，但内部的列表仍然指向同一个对象。

## 怎么判断完成质量

不需要强行要求所有人写出完全相同的代码。更值得观察的是下面四点：

1. **结果是否正确**：输出值和题目要求一致。
2. **思路是否直接**：没有为了简单问题写大量无关代码。
3. **基础是否完整**：会使用常见容器、函数、文件和标准库。
4. **解释是否准确**：最后三题能说明对象引用和状态变化原因，而不是只猜输出。
